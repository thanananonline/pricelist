# Handoff: แดชบอร์ดราคาสินค้า (Thananan)

## Overview
ระบบแสดง/จัดการราคาสินค้าวัสดุก่อสร้าง แบ่งเป็น 2 หน้า:
- **หน้าร้าน.dc.html** — หน้าบ้าน (frontend) สำหรับพนักงาน/ลูกค้าดูราคาสินค้า แบบ read-only
- **แดชบอร์ดราคาสินค้า.dc.html** — หลังบ้าน (backend/admin) สำหรับแก้ไขราคา สต็อก จัดการผู้ใช้ ไฟล์ PDF แบบผลิตภัณฑ์คอนกรีต และข้อมูลติดต่อ

## About the Design Files
ไฟล์ในแพ็กเกจนี้เป็น **ต้นแบบ HTML (design reference)** ที่สร้างขึ้นเพื่อโชว์หน้าตาและการทำงานที่ต้องการ ไม่ใช่โค้ด production ให้ก็อปไปใช้ตรงๆ งานของนักพัฒนาคือ **นำแบบ HTML นี้ไปสร้างใหม่ในโครงสร้างโปรเจกต์จริง** (React/Vue/Next.js หรือ stack ที่ทีมใช้อยู่) โดยใช้ pattern และ library ที่มีอยู่แล้วในโค้ดเบส หรือเลือก framework ที่เหมาะสมถ้ายังไม่มีโปรเจกต์อยู่

ข้อมูลสินค้า/ผู้ใช้/ไฟล์ PDF/ข้อมูลติดต่อในต้นแบบนี้เก็บด้วย `localStorage` ของเบราว์เซอร์เท่านั้น (ไม่มี backend จริง) — ในโปรดักชันต้องเปลี่ยนเป็น API/DB จริงและระบบ auth ที่ปลอดภัย (ต้นแบบนี้ตรวจ user/password แบบ plaintext ใน localStorage เพื่อสาธิตเท่านั้น ห้ามใช้ตรงจริง)

## Fidelity
**High-fidelity (hifi)** — สี ตัวอักษร ระยะห่าง และการโต้ตอบ (ค้นหา/กรอง/เรียง/แก้ไข) ทำงานจริงในต้นแบบ ให้นักพัฒนา recreate ตาม pixel/behavior นี้โดยใช้ design system ของโค้ดเบสจริงถ้ามี

## Screens / Views

### 1. หน้าร้าน (Storefront) — หน้าบ้าน
- **หน้าแรก**: แนะนำร้าน + การ์ดหมวดหมู่สินค้า (9 หมวด: ปูนซีเมนต์, อิฐมวลเบา, กระเบื้องหลังคา, เหล็ก, กระเบื้องปูพื้น/ผนัง, สุขภัณฑ์, ประตู-ไม้, สี, คอนกรีตสำเร็จรูป)
- **รายการสินค้า**: ค้นหา, กรองหมวดหมู่, เรียงราคา, สลับมุมมองกริด/รายการ, แสดงราคาก่อน-หลังส่วนลด, สถานะสต็อก
- **แบบผลิตภัณฑ์คอนกรีต**: แสดงรายการไฟล์ PDF ที่หลังบ้านอัปโหลดไว้ (โหลดจาก localStorage key `thananan_concrete_files`, sync ทุก 2 วินาที + `storage` event)
- **ติดต่อเรา**: แสดงรายชื่อผู้ติดต่อ (ชื่อ/แผนก/เบอร์โทร/ที่อยู่) จาก localStorage key `thananan_contact_info`

### 2. แดชบอร์ดราคาสินค้า (Admin) — หลังบ้าน
- **Login/Register**: username + password, สร้าง/เพิ่มผู้ใช้ได้ (เก็บใน localStorage key `thananan_users`)
- **หน้าราคาสินค้า**: ตารางสินค้าพร้อมค้นหา/กรอง/เรียง, แก้ไข ชื่อ/ราคา/หน่วยนับ/สต็อก แบบ inline, ปุ่ม "+ เพิ่มสินค้า" เปิดฟอร์มกรอก ชื่อ/หมวดหมู่/ราคา/หน่วยนับ/สต็อก — บันทึกลง localStorage key `thananan_custom_products`; การแก้ไขสินค้าเดิมเก็บเป็น override ที่ key `thananan_price_overrides`
- **หน้าแบบผลิตภัณฑ์คอนกรีต**: อัปโหลดไฟล์ PDF (เก็บเป็น base64 data URL ใน localStorage key `thananan_concrete_files`) — ต้องเปลี่ยนเป็นอัปโหลดไป object storage จริงในโปรดักชัน
- **จัดการผู้ใช้**: เพิ่ม/ลบผู้ใช้แอดมิน
- **ติดต่อเรา**: ฟอร์มเพิ่มผู้ติดต่อ (ชื่อ/แผนก/เบอร์โทร/ที่อยู่) เพิ่มได้หลายรายการ, ลบได้ — sync ไปแสดงที่หน้าร้าน

## Interactions & Behavior
- ค้นหา/กรอง/เรียง เป็น client-side filter บน array สินค้าที่รวม RAW_PRODUCTS + customProducts + overrides
- แก้ไขราคา/สต็อก/ชื่อ/หน่วย: กดปุ่มแก้ไข → inline input → "บันทึก" เขียนกลับ localStorage
- หน้าร้านกับหลังบ้าน sync กันผ่าน `localStorage` + `storage` event + polling ทุก 2 วินาที (จำลอง "real-time" โดยไม่มี backend — ในโปรดักชันควรใช้ WebSocket/polling API จริง)

## Design Tokens
- โทนสีหลัก: กรมท่า/น้ำเงินเข้ม `oklch(0.32 0.14 275)` (ปุ่ม/แอคเซนต์หลัก), `oklch(0.38 0.15 275)` (ราคา/CTA), `oklch(0.26 0.13 275)` และ `oklch(0.2 0.12 275)` (แถบนำทาง/พื้นเข้ม) — อ้างอิงโทนจากเว็บ thanananonline.com
- พื้นหลังหน้า: `oklch(0.975 0.006 70)` (ครีมอ่อน)
- ตัวอักษร: `'Noto Sans Thai', 'Kanit', sans-serif`
- ปุ่ม/การ์ด border-radius: 9–14px

## Assets
ไม่มีรูปภาพ/ไอคอนจากภายนอก ใช้ emoji และบล็อกสีไล่เฉด (oklch) แทนรูปสินค้า — ควรเปลี่ยนเป็นรูปสินค้าจริงในโปรดักชัน

## Files
- `หน้าร้าน.dc.html` / `หน้าร้าน (standalone).html` — หน้าบ้าน
- `แดชบอร์ดราคาสินค้า.dc.html` / `แดชบอร์ดราคาสินค้า (standalone).html` — หลังบ้าน

ไฟล์ `(standalone).html` เปิดได้ตรงในเบราว์เซอร์โดยไม่ต้องมี dependency อื่น ใช้สำหรับดูงานอย่างรวดเร็ว ส่วนไฟล์ `.dc.html` คือไฟล์ต้นทางที่มีโครงสร้าง component
